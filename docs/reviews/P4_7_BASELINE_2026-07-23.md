# P4.7-0 Baseline Observation 2026-07-23

> Document status: partial P4.7-0 observation, not a performance gate.
> The formal baseline still requires full-column traversal, cache counters, allocation measurement and four-combination JFR.

## Fixture

- Profile: non-persisted P4.6 archipelago smoke profile.
- Seed: `123456789`.
- Window: 16 x 16 columns beginning at `(8192, 8192)`.
- Runtime path: `OUTER_CONTINENTS`, `WITH_FLOOR`, `REGION_PLANNED`, archipelago enabled.
- Operation: `EndHeightmap.sampleTerrainProfile` into one caller-owned `EndTerrainProfileBuffer`.
- Warmup: 1 chunk ordered + 1 chunk deterministic shuffled.
- Measurement: 4 chunks per traversal, 1024 profile samples.

## Observation

One local JDK 21 run produced:

| Traversal | Time |
| --- | ---: |
| Ordered columns | 60,385.9 ns/profile |
| Deterministic shuffled columns | 32,033.3 ns/profile |

Both traversals produced the same checksum: `2755942566284`.

The checksum equality is now a test assertion. The timing difference is evidence for further cache, branch and JIT investigation; it is not a claim that shuffled access is intrinsically faster.

## Limits

This observation does not measure full density columns, raw-top evaluation counts, cache hit/miss/collision/eviction, allocations, p50/p95, tile construction, client rendering or ETF/RTF/C2ME JFR combinations. P4.7-0 remains open until those measurements exist.
